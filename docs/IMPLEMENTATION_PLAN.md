# Implementation Plan — Mobile Fit App

> Iterative, complexity-progressive roadmap for building the full stack while learning DevOps hands-on.  
> Each iteration ships something real and runnable before moving on to the next layer.

---

## How to Use This Plan

- Work **one iteration at a time**. Do not skip ahead.
- Each iteration ends with a working, verifiable checkpoint.
- Skills learned are listed per iteration so you can track your progress.
- Estimated time is loose — quality over speed.

---

## Iteration 1 — Monorepo, Backend Skeleton & Docker Compose

**Goal:** Get the project running locally with a single `make dev` command.

**What you build:**
- Monorepo folder structure (as defined in ARCHITECTURE.md)
- Git repository initialized with `.gitignore` and `.env.example`
- FastAPI backend with a single `/health` endpoint
- PostgreSQL container via Docker Compose
- `Makefile` with `dev`, `dev-down`, `dev-logs` targets

**Tasks:**
- [ ] Initialize git repo and commit initial structure
- [ ] Create `docker-compose.yaml` with `postgres` and `api` services
- [ ] Scaffold `backend/` with `app/main.py`, `app/config.py`, `app/routers/health.py`
- [ ] Write `backend/Dockerfile` (multi-stage, non-root user)
- [ ] Write `backend/pyproject.toml` and lock with `uv`
- [ ] Add `.env.example` with all required variables documented
- [ ] Create `Makefile` with basic dev targets
- [ ] Verify: `make dev` → `curl http://localhost:8080/health` returns `{"status":"ok"}`

**Skills practiced:**
- Docker & Docker Compose
- FastAPI project layout
- uv for Python dependency management
